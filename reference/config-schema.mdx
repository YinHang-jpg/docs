The `review/examples/run_example.py` script reads a single JSON configuration file that controls the simulation length, model paths, KDE visualization parameters, and output file name. The bundled default lives at `review/examples/demo_input.json`. You can point the script at any conforming JSON file with the `--config` flag.

```bash
# Run with the default config and show the plot interactively
python review/examples/run_example.py --plot

# Run with a custom config; abort if PINN/ weights are missing
python review/examples/run_example.py --config my_config.json --skip-train
```

---

## Complete default file

```json
{
  "_comment": "Minimal input configuration for review/examples/run_example.py.",
  "_pipeline": "PINN simulation via results/PD_vs_time/PD_time.py for 0.01 s, then a single-time PINN/DEM/FEM comparison plot.",

  "n_steps": 10000,
  "dt_s": 1e-6,
  "target_time_s": 0.01,

  "pinn_model_base_path": null,

  "x_window_mm": [2.0, 32.0],
  "kde_sigma_mm": 0.3,
  "kde_n_grid": 200,

  "pinn_csv": "results/PD_vs_time/pinn_particle_positions.csv",
  "dem_csv":  "results/PD_vs_time/dem_particle_distribution_data.csv",
  "fem_csv":  "validation/comsol_positions.csv",
  "dem_column": "relative_change_1.0e+04",

  "output_figure": "density_comparison_t0.01s.png"
}
```

All paths are resolved relative to the repository root unless they are absolute.

## Field reference

### Simulation timing

<ParamField path="n_steps" type="int" required>
  Total number of Euler integration timesteps to advance the PINN simulation. The simulated physical time equals `n_steps × dt_s`. For the default values this is `10 000 × 1 µs = 0.01 s`.

  Increasing `n_steps` extends the simulated duration linearly. Wall-clock cost scales proportionally; see the [Benchmarks](/reference/benchmarks) page for measured runtimes.
</ParamField>

<ParamField path="dt_s" default="1e-6" type="float">
  Integration timestep in seconds. The default 1 µs is stable for 2 µm silica particles in a 10 kHz standing wave and is the value used in all published benchmarks. Using a larger timestep may cause velocity blow-up in the early transient when forces are strongest.
</ParamField>

<ParamField path="target_time_s" default="0.01" type="float">
  Physical time in seconds at which the density-comparison snapshot is taken. `run_example.py` selects the PINN position snapshot whose time is closest to this value. The FEM CSV is also queried for the row nearest `target_time_s`.

  This field also sets the `position_interval` passed to `PD_time.run_single_simulation`, so the intermediate snapshot list always includes a point near `target_time_s`.
</ParamField>

---

### Model weights

<ParamField path="pinn_model_base_path" default="null" type="string | null">
  Path to the directory that contains the ten PINN artifacts (five `.pth` weight files and five `_normalization_params.json` files). Relative paths are resolved from the repository root.

  Set to `null` (or omit the key entirely) to use the default `PINN/` directory at the repository root. Supply an explicit path only when you want to switch to a different trained pack — for example a model trained at a different acoustic frequency:

  ```json
  "pinn_model_base_path": "PINN/freq_8k"
  ```

  The script validates that the target directory contains all ten artifacts before starting the simulation. If any are missing and `--skip-train` is **not** passed, it automatically invokes the five training scripts to fill `PINN/` (not the custom path).

  <Warning>
    The bundled trainers only write to the default `PINN/` directory. If you specify a custom `pinn_model_base_path`, you must populate it manually by copying or symlinking trained artifacts.
  </Warning>
</ParamField>

---

### KDE visualization window

<ParamField path="x_window_mm" default="[2.0, 32.0]" type="[float, float]">
  Analysis window `[x_min, x_max]` in millimeters used for the density-comparison plot. Particles and KDE values outside this range are excluded from the figure. The default `[2.0, 32.0]` mm trims the 2 mm edge regions where boundary effects can distort KDE estimates.
</ParamField>

<ParamField path="kde_sigma_mm" default="0.3" type="float">
  Gaussian kernel bandwidth (standard deviation) for the KDE density estimate, in millimeters. Smaller values produce sharper, noisier curves; larger values smooth out particle clustering. The validation paper uses σ = 0.2 mm for the fine-grid COMSOL comparison and σ = 0.3 mm for run-time demos.
</ParamField>

<ParamField path="kde_n_grid" default="200" type="int">
  Number of equally spaced grid points spanning `[0, 34]` mm at which the KDE density is evaluated before windowing by `x_window_mm`. A higher value produces a smoother curve at the cost of slightly more computation.
</ParamField>

---

### Reference data paths

<ParamField path="pinn_csv" default="results/PD_vs_time/pinn_particle_positions.csv" type="string">
  Path (relative to the repository root) to the CSV file where `PD_time.run_single_simulation` writes PINN particle positions over time. This file is created automatically by the simulation run; you do not need to provide it in advance.
</ParamField>

<ParamField path="dem_csv" default="results/PD_vs_time/dem_particle_distribution_data.csv" type="string">
  Path to the pre-computed DEM relative concentration change CSV. This file is generated by `results/PD_vs_time/PD_vs_time_DEM.py`. The script raises a `FileNotFoundError` with a hint if the file is absent.
</ParamField>

<ParamField path="fem_csv" default="validation/comsol_positions.csv" type="string">
  Path to the COMSOL FEM particle-position CSV exported from the acoustofluidic simulation. Each row corresponds to one time snapshot; columns beyond the first are per-particle x-positions. The script finds the row whose time is nearest `target_time_s`.
</ParamField>

<ParamField path="dem_column" default="relative_change_1.0e+04" type="string">
  Column name in `dem_csv` that holds the relative concentration change at the particle count used in the benchmark. The default `"relative_change_1.0e+04"` corresponds to N = 10 000 particles. Run `python -c "import pandas as pd; print(pd.read_csv('results/PD_vs_time/dem_particle_distribution_data.csv').columns.tolist())"` to see all available columns.
</ParamField>

---

### Output

<ParamField path="output_figure" default="density_comparison_t0.01s.png" type="string">
  Filename for the saved comparison figure. The file is written to the same directory as the config file (`review/examples/` when using the default). Pass `--plot` to also display the figure interactively in a Matplotlib window.
</ParamField>

## Passing a custom config

Create a JSON file with any subset of fields you want to override, then point `run_example.py` at it:

```json
{
  "n_steps": 50000,
  "dt_s": 1e-6,
  "target_time_s": 0.05,
  "pinn_model_base_path": null,
  "x_window_mm": [2.0, 32.0],
  "kde_sigma_mm": 0.2,
  "kde_n_grid": 300,
  "pinn_csv": "results/PD_vs_time/pinn_particle_positions.csv",
  "dem_csv":  "results/PD_vs_time/dem_particle_distribution_data.csv",
  "fem_csv":  "validation/comsol_positions.csv",
  "dem_column": "relative_change_1.0e+04",
  "output_figure": "density_comparison_t0.05s.png"
}
```

```bash
python review/examples/run_example.py \
    --config my_config.json \
    --plot
```

All eleven required keys must be present (`_comment` and `_pipeline` fields are optional and ignored by the loader). The script raises a `KeyError` listing any missing keys if validation fails.
