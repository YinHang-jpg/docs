FF-PINN exposes a small, focused public API. You initialize a particle ensemble, load the five factorized sub-network checkpoints into a unified model object, then call that model every integration step to obtain physical forces. This page documents each function and class in the order you typically call them.

---

## `initialize_particles`

```python
from initialization.particle_initialization import initialize_particles
```

Creates an ensemble of aerosol particles in a 2-D rectangular domain, returning NumPy arrays that are ready to pass directly to the PyTorch integration loop.

```python
def initialize_particles(
    N: int = 0,
    domain_size: tuple[float, float] = (0.01, 0.01),
    diameter: float = 2e-6,
    density: float = 2000,
    init_mode: str = "linear",
) -> tuple[np.ndarray, np.ndarray, np.ndarray, np.ndarray]:
```

### Parameters

<ParamField path="N" type="int" default="0">
  Number of particles to place. For `init_mode='random'` the domain must be large enough to satisfy the minimum-separation constraint (ÔëÑ 2 ┬Ám center-to-center); an attempt limit of `max(1 000 ├ù N, 100 000)` is enforced and a `RuntimeError` is raised if placement fails.
</ParamField>

<ParamField path="domain_size" type="tuple[float, float]" default="(0.01, 0.01)">
  `(Lx, Ly)` in meters. The default 10 mm ├ù 10 mm domain is suitable for small validation runs. Production runs typically use `(0.034, 0.034)` (34 mm square, one full acoustic wavelength at 10 kHz).
</ParamField>

<ParamField path="diameter" type="float" default="2e-6">
  Nominal particle diameter in meters. Ignored when `init_mode='linear_poly'`, which draws per-particle diameters uniformly from [0.1 ┬Ám, 10 ┬Ám].
</ParamField>

<ParamField path="density" type="float" default="2000">
  Material density in kg/m┬│. Used to compute mass as `¤ü ┬À (4/3)¤Ç r┬│`.
</ParamField>

<ParamField path="init_mode" type="str" default="linear">
  Spatial layout strategy. Valid values:

  | Value | Description |
  |---|---|
  | `"linear"` | Uniform spacing along x at y = Ly/2. |
  | `"linear_poly"` | Same as `"linear"` but with polydisperse diameters drawn from U[0.1, 10] ┬Ám. |
  | `"random"` | Poisson-disk-like rejection sampling; guarantees ÔëÑ 2 ┬Ám center-to-center separation. Requires `scipy` for the final validation pass. |
  | `"uniform"` | Near-uniform grid (ÔêÜN ├ù ÔêÜN), with extra random points added if N is not a perfect square. |
  | `"sample"` | Two vertical stacks placed at x = 8.5 mm and x = 25.5 mm; useful for reproducing the paper's benchmark geometry. |
  | `"normal"` | Truncated-Gaussian quantile placement along x (¤â = Lx / 10), all at y = 0.01 m. |
</ParamField>

### Returns

<ResponseField name="positions" type="np.ndarray, shape (N, 2)">
  Initial particle positions `[x, y]` in meters.
</ResponseField>

<ResponseField name="velocities" type="np.ndarray, shape (N, 2)">
  Initial velocities, always zero.
</ResponseField>

<ResponseField name="radii" type="np.ndarray, shape (N,)">
  Per-particle radii in meters (`diameter / 2` for monodisperse modes).
</ResponseField>

<ResponseField name="mass" type="np.ndarray, shape (N,)">
  Per-particle mass in kg, computed as `density ┬À (4/3)¤Ç r┬│`.
</ResponseField>

### Example

```python
import torch
from initialization.particle_initialization import initialize_particles

positions, velocities, radii, mass = initialize_particles(
    N=10_000,
    domain_size=(0.034, 0.034),  # 34 mm square
    diameter=2e-6,               # 2 ┬Ám silica spheres
    density=2000,                # kg/m┬│
    init_mode="linear",
)

# Convert to tensors for the integration loop
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
pos_t  = torch.as_tensor(positions,  dtype=torch.float32, device=device)
vel_t  = torch.as_tensor(velocities, dtype=torch.float32, device=device)
rad_t  = torch.as_tensor(radii,      dtype=torch.float32, device=device)
mass_t = torch.as_tensor(mass,       dtype=torch.float32, device=device)

print(f"Initialized {len(positions)} particles")
print(f"x range: {positions[:, 0].min()*1000:.2f} ÔÇô {positions[:, 0].max()*1000:.2f} mm")
```

## `load_individual_models`

```python
from mechanisms.UNIFIED_PINN import load_individual_models
```

Loads the five factorized sub-network checkpoints (`.pth` files) and their companion JSON normalization files from a model directory and returns them in a single dict. The function infers the acoustic frequency from a `freq_<N>k` substring in the path (e.g. `PINN/freq_10k` ÔåÆ 10 000 Hz) and falls back to the global `frequency` constant otherwise.

```python
def load_individual_models(
    device: str = "cpu",
    model_base_path: str = "PINN",
) -> dict:
```

### Parameters

<ParamField path="device" type="str" default="'cpu'">
  PyTorch device string. Pass `"cuda"` to map weights directly to GPU memory.
</ParamField>

<ParamField path="model_base_path" type="str" default="'PINN'">
  Directory that contains all ten artifacts: five `.pth` weight files and five `_normalization_params.json` files. Missing files are reported to stdout but do not raise an exception ÔÇö the caller should validate the returned dict before proceeding.
</ParamField>

### Returns

<ResponseField name="models" type="dict">
  Dictionary with up to ten keys. Use it to build a `UnifiedNormalizer` and then a `PhysicalUnifiedModel`.

  | Key | Type | Contents |
  |---|---|---|
  | `"arf_x"` | `ARFNetX` | Spatial factor network for acoustic radiation force |
  | `"arf_t"` | `ARFNetT` | Temporal factor network for acoustic radiation force |
  | `"stokes_x"` | `StokesNetX` | Spatial factor for Stokes drag / acoustic streaming |
  | `"stokes_t"` | `StokesNetT` | Temporal factor for Stokes drag |
  | `"stokes_v"` | `StokesNetV` | Velocity factor for Stokes drag |
  | `"arf_x_norm"` | `ARFNormalizerX` | Normalization helper for ARF x branch |
  | `"arf_t_norm"` | `ARFNormalizerT` | Normalization helper for ARF t branch |
  | `"stokes_x_norm"` / `"stokes_t_norm"` / `"stokes_v_norm"` | normalizer objects | Companions for the three Stokes branches |
</ResponseField>

### Example

```python
from mechanisms.UNIFIED_PINN import load_individual_models, UnifiedNormalizer
import os

device = "cuda"
model_base_path = "PINN"

models = load_individual_models(device=device, model_base_path=model_base_path)

required = ["arf_x", "arf_t", "stokes_x", "stokes_t", "stokes_v"]
missing  = [k for k in required if k not in models]
if missing:
    raise RuntimeError(f"Incomplete checkpoint pack; missing: {missing}")

print("All five sub-networks loaded.")
```

## `UnifiedNormalizer`

```python
from mechanisms.UNIFIED_PINN import UnifiedNormalizer
```

Aggregates per-branch normalization statistics (mean, standard deviation, and input ranges) that were saved during training into a single object. `PhysicalUnifiedModel` depends on it for inference-time input scaling and output de-normalization.

### `load_from_individual_models`

```python
def load_from_individual_models(
    self,
    model_paths: dict[str, str],
    device: str = "cpu",
) -> None:
```

Reads five JSON normalization files and populates `self.stats` with the following keys: `x_min`, `x_max`, `t_min`, `t_max`, `vx_min`, `vx_max`, and per-branch `mu` / `sigma` pairs for each output factor.

<ParamField path="model_paths" type="dict[str, str]">
  Dictionary mapping five logical keys to absolute or relative file paths:

  ```python
  {
      "arf_x_norm":    "PINN/arf_model_x_normalization_params.json",
      "arf_t_norm":    "PINN/arf_model_t_normalization_params.json",
      "stokes_x_norm": "PINN/stokes_model_x_normalization_params.json",
      "stokes_t_norm": "PINN/stokes_model_t_normalization_params.json",
      "stokes_v_norm": "PINN/stokes_model_v_normalization_params.json",
  }
  ```

  Missing files are skipped with a fallback to sensible defaults, but you should ensure all five are present for correct inference.
</ParamField>

<ParamField path="device" type="str" default="'cpu'">
  Kept for API compatibility; JSON stats are stored as plain Python floats.
</ParamField>

### Normalization helpers

After calling `load_from_individual_models`, three helper methods become available:

| Method | Purpose |
|---|---|
| `normalize_inputs(x, vx, t)` | Maps raw tensors to [0, 1] with periodic wrapping for x and t (ARF windows). |
| `normalize_inputs_stokes(x, vx, t)` | Same but uses Stokes-specific x/t windows. |
| `denormalize_arf_force(fx_norm, ft_norm)` | Converts network outputs back to physical force scale for ARF factors. |
| `denormalize_stokes_force(fx_norm, ft_norm, fv_norm)` | Same for all three Stokes factors. |

### Example

```python
import os
from mechanisms.UNIFIED_PINN import UnifiedNormalizer

unified_norm = UnifiedNormalizer()
base = "PINN"
unified_norm.load_from_individual_models(
    {
        "arf_x_norm":    os.path.join(base, "arf_model_x_normalization_params.json"),
        "arf_t_norm":    os.path.join(base, "arf_model_t_normalization_params.json"),
        "stokes_x_norm": os.path.join(base, "stokes_model_x_normalization_params.json"),
        "stokes_t_norm": os.path.join(base, "stokes_model_t_normalization_params.json"),
        "stokes_v_norm": os.path.join(base, "stokes_model_v_normalization_params.json"),
    },
    device="cpu",
)
print("x window:", unified_norm.stats["x_min"], "ÔåÆ", unified_norm.stats["x_max"], "m")
print("t window:", unified_norm.stats["t_min"], "ÔåÆ", unified_norm.stats["t_max"], "s")
```

## `PhysicalUnifiedModel`

```python
from mechanisms.UNIFIED_PINN import PhysicalUnifiedModel
```

The primary inference object. It wraps the five sub-networks and the `UnifiedNormalizer` and exposes a single `forward` call that returns a per-particle force tensor in Newtons. All sub-network forwards run under `torch.no_grad()`.

```python
class PhysicalUnifiedModel(nn.Module):
    def __init__(
        self,
        models: dict,
        unified_norm: UnifiedNormalizer,
        device: str = "cpu",
    ) -> None:
```

### Constructor parameters

<ParamField path="models" type="dict">
  The dict returned by `load_individual_models`. Must contain at minimum the five keys `"arf_x"`, `"arf_t"`, `"stokes_x"`, `"stokes_t"`, and `"stokes_v"`.
</ParamField>

<ParamField path="unified_norm" type="UnifiedNormalizer">
  A `UnifiedNormalizer` instance that has already been populated via `load_from_individual_models`. The model infers the acoustic frequency from `unified_norm.stats["t_max"]` and prints a sanity-check summary (frequency, wavelength, nodal spacing) on construction.
</ParamField>

<ParamField path="device" type="str" default="'cpu'">
  Computation device. Pass `"cuda"` for GPU inference. Input tensors are automatically moved to this device inside `forward`.
</ParamField>

### `forward`

```python
@torch.no_grad()
def forward(
    self,
    x: torch.Tensor,    # shape (N,), particle x-positions [m]
    vx: torch.Tensor,   # shape (N,), particle x-velocities [m/s]
    t: torch.Tensor,    # shape (N,), current time [s] (usually a constant vector)
) -> torch.Tensor:      # shape (N,), total x-force [N]
```

Evaluates the combined acoustic radiation force and Stokes drag for all N particles in a single batched pass. The y-component of force is not returned; FF-PINN is a 1-D force model along the acoustic propagation axis.

### Returns

<ResponseField name="total_fx" type="torch.Tensor, shape (N,)">
  Total x-direction force on each particle in Newtons. Positive values push particles toward increasing x; in a standing-wave field particles migrate toward pressure nodes.
</ResponseField>

### Full assembly and call example

```python
import os
import torch
from mechanisms.UNIFIED_PINN import (
    load_individual_models,
    UnifiedNormalizer,
    PhysicalUnifiedModel,
)
from initialization.particle_initialization import initialize_particles

# --- build model ---
device = "cuda" if torch.cuda.is_available() else "cpu"
base   = "PINN"

models = load_individual_models(device=device, model_base_path=base)

unified_norm = UnifiedNormalizer()
unified_norm.load_from_individual_models(
    {k: os.path.join(base, f) for k, f in {
        "arf_x_norm":    "arf_model_x_normalization_params.json",
        "arf_t_norm":    "arf_model_t_normalization_params.json",
        "stokes_x_norm": "stokes_model_x_normalization_params.json",
        "stokes_t_norm": "stokes_model_t_normalization_params.json",
        "stokes_v_norm": "stokes_model_v_normalization_params.json",
    }.items()},
)
models["unified_norm"] = unified_norm

phys_model = PhysicalUnifiedModel(models, unified_norm, device=device)

# --- initialize particles and run one step ---
positions, velocities, radii, mass = initialize_particles(
    N=10_000, domain_size=(0.034, 0.034), init_mode="linear"
)
x_t  = torch.tensor(positions[:, 0],  dtype=torch.float32, device=device)
vx_t = torch.tensor(velocities[:, 0], dtype=torch.float32, device=device)
t_t  = torch.zeros_like(x_t)  # t = 0 s

forces = phys_model(x_t, vx_t, t_t)  # ÔåÆ shape (10_000,)
print(f"Force range: [{forces.min():.3e}, {forces.max():.3e}] N")
```

## `compute_unified_force`

```python
from mechanisms.UNIFIED_PINN import compute_unified_force
```

A legacy NumPy-in / NumPy-out helper that wraps `PhysicalUnifiedModel` for callers that work with plain arrays instead of tensors. The function expects that `models["unified_norm"]` is already populated (see example above).

```python
def compute_unified_force(
    positions: np.ndarray,   # (N, 2), x/y positions [m]
    velocities: np.ndarray,  # (N, 2), x/y velocities [m/s]
    time: float,             # current simulation time [s]
    models: dict,            # dict from load_individual_models, with "unified_norm" key
    device: str = "cpu",
) -> np.ndarray:             # (N, 2)
```

### Parameters

<ParamField path="positions" type="np.ndarray, shape (N, 2)">
  Particle positions in meters. Only the x-column (`positions[:, 0]`) is used for force calculation.
</ParamField>

<ParamField path="velocities" type="np.ndarray, shape (N, 2)">
  Particle velocities in m/s. Only `velocities[:, 0]` enters the Stokes drag term.
</ParamField>

<ParamField path="time" type="float">
  Current physical simulation time in seconds. Broadcast to all N particles internally.
</ParamField>

<ParamField path="models" type="dict">
  The same dict built with `load_individual_models`, with the additional `"unified_norm"` key set to a populated `UnifiedNormalizer`.
</ParamField>

<ParamField path="device" type="str" default="'cpu'">
  Compute device for the internal tensor operations.
</ParamField>

### Returns

<ResponseField name="forces" type="np.ndarray, shape (N, 2)">
  Column 0 is the x-direction force (ARF + Stokes) in Newtons. Column 1 is always zero (no y-force in this model).
</ResponseField>

### Example

```python
import numpy as np
from initialization.particle_initialization import initialize_particles
from mechanisms.UNIFIED_PINN import load_individual_models, UnifiedNormalizer, compute_unified_force
import os

base = "PINN"
models = load_individual_models(device="cpu", model_base_path=base)
unified_norm = UnifiedNormalizer()
unified_norm.load_from_individual_models(
    {k: os.path.join(base, v) for k, v in {
        "arf_x_norm":    "arf_model_x_normalization_params.json",
        "arf_t_norm":    "arf_model_t_normalization_params.json",
        "stokes_x_norm": "stokes_model_x_normalization_params.json",
        "stokes_t_norm": "stokes_model_t_normalization_params.json",
        "stokes_v_norm": "stokes_model_v_normalization_params.json",
    }.items()},
)
models["unified_norm"] = unified_norm

positions, velocities, radii, mass = initialize_particles(N=500, init_mode="linear")

forces = compute_unified_force(positions, velocities, time=0.0, models=models, device="cpu")
print(f"forces.shape = {forces.shape}")        # (500, 2)
print(f"Fx range: [{forces[:, 0].min():.2e}, {forces[:, 0].max():.2e}] N")
```

## `compute_pressure_gradient_and_apply_arf`

```python
from mechanisms.ARF import compute_pressure_gradient_and_apply_arf
```

An analytical ARF baseline that computes the acoustic radiation force directly from the standing-wave pressure field without any neural network. Use this to sanity-check PINN predictions or to run a reference simulation when trained weights are unavailable.

The force on each particle follows the front/back pressure-difference formula:

```
F_p = ¤Ç d_p┬▓ [p(x ÔêÆ ÔêÜ2 d_p/4, t) ÔêÆ p(x + ÔêÜ2 d_p/4, t)] / 4
```

where the pressure field is evaluated analytically from the standing-wave solution.

```python
def compute_pressure_gradient_and_apply_arf(
    positions: np.ndarray,  # (N, 2) [m]
    radii: np.ndarray,      # (N,)   [m]
    time: float,            # current time [s]
) -> np.ndarray:            # (N, 2) [N]
```

### Parameters

<ParamField path="positions" type="np.ndarray, shape (N, 2)">
  Particle x/y positions in meters. The analytical pressure field is 1-D (varies only with x), so the y-component is ignored.
</ParamField>

<ParamField path="radii" type="np.ndarray, shape (N,)">
  Per-particle radii in meters. The particle diameter `d_p = 2r` sets the sampling offset `ÔêÜ2 d_p/4` around each particle center.
</ParamField>

<ParamField path="time" type="float">
  Evaluation time in seconds. The cosine time factor `cos(¤ët)` is applied analytically.
</ParamField>

### Returns

<ResponseField name="arf_force" type="np.ndarray, shape (N, 2)">
  Column 0 is the x-direction ARF in Newtons. Column 1 is always zero. This function does **not** include Stokes drag.
</ResponseField>

### Example

```python
import numpy as np
from mechanisms.ARF import compute_pressure_gradient_and_apply_arf
from initialization.particle_initialization import initialize_particles

positions, _, radii, _ = initialize_particles(N=200, domain_size=(0.034, 0.034))

arf = compute_pressure_gradient_and_apply_arf(positions, radii, time=0.0)
print(f"ARF x range: [{arf[:, 0].min():.2e}, {arf[:, 0].max():.2e}] N")
```

<Note>
  This function iterates over all N particles in a Python loop and scales as O(N). It is intended for validation and small-N checks. For production simulations use `PhysicalUnifiedModel` or `compute_unified_force`, which batch all N particles through the network in a single tensor operation.
</Note>
